## Drafting Instructions

1. Read `style-guide.md` + `content-lessons.md`
2. Write tweet 1 first â€” spend the most time here. It determines whether anyone reads the rest.
3. Outline the thread as a list of one-line ideas before writing full tweets
4. Each tweet = one point. If a point needs two tweets, it's too complex â€” simplify or split
5. Include at least 2-3 tweets that are individually screenshot-worthy (a stat, a punchline, a framework)
6. The thread should feel like a conversation, not a lecture
7. Verify each tweet is under 280 characters
